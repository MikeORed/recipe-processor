# Project Structure

## Architecture: Hexagonal (Ports & Adapters)

```
heirloom/
├── bin/                            # CLI entry point
├── infra/                          # CDK stacks
│   ├── stateful/                   # S3 bucket, DynamoDB tables (RETAIN policy)
│   └── stateless/                  # IAM policies, Textract/Bedrock access
├── src/
│   ├── domain/                     # Pure business logic, zero external deps
│   │   ├── ports/                  # Interfaces: OCRProvider, StructureExtractor, DataStore, Exporter
│   │   ├── models/                 # Zod schemas + types: Recipe, Source, Job
│   │   └── services/               # Ingest, transcribe, export orchestration
│   ├── adapters/
│   │   ├── inbound/                # CLI command handlers
│   │   └── outbound/               # S3, Textract, Bedrock, DynamoDB, PDF, Obsidian adapters
│   ├── shared/                     # Logger, custom errors, shared schemas
│   └── config/                     # Convict config (AWS region, table names, bucket, etc.)
├── jobs/                           # ⛔ .gitignored — local job landing zones
│   └── <job-name>/
│       ├── images/                 # User drops source photos here
│       └── manifest.csv            # Generated by ingest, edited by user
├── test/                           # Integration tests
└── .kiro/
    ├── specs/                      # Feature specs
    └── steering/                   # Steering docs (this directory)
```

## Dependency Rules (strict)

- `domain/` → depends on **nothing**. No AWS SDK, no file system, no adapters.
- `adapters/` → depends on `domain/` (implements port interfaces). Only layer that touches AWS SDK or file system.
- `shared/` → depends on nothing. Consumed by all layers.
- `config/` → depends on nothing. Consumed by all layers.
- `infra/` → depends on `src/` only for reference constants (table names, bucket names).

## Test File Placement

Tests are colocated with source files:
- `foo.unit.ts` — unit test for `foo.ts`
- `foo.pbt.ts` — property-based test for `foo.ts`

Integration/cross-stack tests live in `test/`.

## CDK Stacks

- **Stateful** (`infra/stateful/`): S3 bucket, DynamoDB tables (Recipes + Jobs). RETAIN policy.
- **Stateless** (`infra/stateless/`): IAM policies, Bedrock inference profile.
